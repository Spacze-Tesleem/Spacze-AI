# Spacze AI — Agent Guide

This file tells AI coding agents how this repository is structured, how to run it, and what conventions to follow.

## Tech stack

- **Runtime**: Node.js 24, TypeScript 5.9, ESM throughout
- **Package manager**: pnpm (workspaces). Do not use npm or yarn.
- **Frontend**: React 19, Vite 7, Tailwind CSS 4, Radix UI, TanStack Query, Wouter
- **API**: Express 5, Pino logging, Zod validation, Drizzle ORM
- **Database**: PostgreSQL 16 (local via Docker), Drizzle ORM + drizzle-zod
- **AI**: OpenAI-compatible SDK pointed at OpenRouter by default
- **API codegen**: Orval generates `@workspace/api-client-react` and `@workspace/api-zod` from `lib/api-spec/openapi.yaml`
- **Testing**: Vitest (API server only)

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `AI_INTEGRATIONS_OPENAI_API_KEY` | Yes | OpenRouter API key |
| `AI_INTEGRATIONS_OPENAI_BASE_URL` | No | API base URL (default: `https://openrouter.ai/api/v1`) |
| `AI_CHAT_MODEL` | No | Model identifier (default: `google/gemma-3-27b-it:free`) |
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `CORS_ALLOWED_ORIGINS` | No | Comma-separated list of allowed CORS origins |
| `PORT` | No | API server port (default: 8080) |

On Ona/Gitpod, set `AI_INTEGRATIONS_OPENAI_API_KEY` as an environment secret — it is injected automatically.

## Key commands

```bash
# Install all workspace dependencies
pnpm install

# Typecheck all packages
pnpm run typecheck

# Build all packages
pnpm run build

# Run API server tests
pnpm --filter @workspace/api-server test

# Push DB schema to local Postgres (dev only)
pnpm --filter @workspace/db push

# Regenerate API client hooks and Zod schemas from openapi.yaml
pnpm --filter @workspace/api-spec codegen

# Start API server in dev mode (requires DATABASE_URL and AI key)
pnpm --filter @workspace/api-server dev

# Start frontend dev server
pnpm --filter @workspace/spacze dev
```

## API design

- All routes are prefixed with `/api`.
- Request bodies and path params are validated with Zod schemas from `@workspace/api-zod`.
- Validation errors return `400 { error: "Validation failed", issues: [...] }`.
- Not-found errors return `404 { error: "..." }`.
- The global error handler in `artifacts/api-server/src/middlewares/error-handler.ts` catches all unhandled errors and never leaks stack traces.
- Three streaming endpoints use SSE (`text/event-stream`): `POST /api/openai/conversations/:id/messages`, `POST /api/projects/:id/generate`, `POST /api/projects/:id/debug`, `POST /api/projects/:id/run`. Each emits `data: {...}\n\n` chunks and ends with `data: {"done":true}\n\n`.

## Adding a new API endpoint

1. Add the route to `lib/api-spec/openapi.yaml`.
2. Run `pnpm --filter @workspace/api-spec codegen` to regenerate Zod schemas and React hooks.
3. Implement the route handler in `artifacts/api-server/src/routes/`.
4. Register the router in `artifacts/api-server/src/routes/index.ts`.
5. Add tests in `artifacts/api-server/src/__tests__/`.

---

## Database changes

1. Edit the schema in `lib/db/src/schema/`.
2. Export new tables/types from `lib/db/src/schema/index.ts`.
3. Run `pnpm --filter @workspace/db push` to apply changes to the local dev database.
4. Update `lib/api-spec/openapi.yaml` if the change affects the API surface.

## Code conventions

- All packages use ESM (`"type": "module"`).
- TypeScript strict mode is on. Do not use `any` without a comment explaining why.
- Imports within a package use `.js` extensions (required for Node ESM).
- Zod schemas are the single source of truth for request/response shapes — do not duplicate validation logic.
- Route handlers must not `throw` — unhandled promise rejections are caught by Express 5's async error propagation and forwarded to the error handler.
- Do not hardcode model names; read from `process.env.AI_CHAT_MODEL`.
- Do not add `console.log` — use the Pino logger (`import { logger } from "../lib/logger"`).

---

## Testing conventions

- Tests live in `artifacts/api-server/src/__tests__/`.
- Use Vitest. Run with `pnpm --filter @workspace/api-server test`.
- Mock `@workspace/db` and `@workspace/integrations-openai-ai-server` in every test file — never hit a real database or AI API in tests.
- Use `vi.resetModules()` in `afterEach` when tests set `process.env` variables, because `app.ts` reads env at module-evaluation time.
- There are currently no frontend tests. New frontend logic should be tested if it contains non-trivial business logic.

---

## Security notes

- `pnpm-workspace.yaml` enforces `minimumReleaseAge: 1440` (1 day) for all packages. Do not disable this.
- CORS is controlled by `CORS_ALLOWED_ORIGINS`. Do not set `origin: true` or `origin: "*"` in production.
- Never log or expose `AI_INTEGRATIONS_OPENAI_API_KEY` or `DATABASE_URL`.
